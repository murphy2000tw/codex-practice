# 方案 C Batch 04 閱讀單字候選清單

## 1. 本批任務目的

本批任務定位為「閱讀文章候選詞整理」，只從現有日文閱讀文章中整理下一批可能適合加入正式日文單字庫的候選詞，供後續人工確認使用。本次不正式匯入任何單字，未修改 `vocabulary.json`、日文閱讀頁、日文單字頁、日文測驗頁或英文頁面。

本次開始前確認：

* `vocabulary.json` 目前為 3156 筆。
* `docs/japanese-plan-c-batch03-post-import-check.md` 存在，並記錄 Batch 03 匯入後功能檢查已完成。
* `docs/reading-vocabulary-import-batch-03.md` 已有「Batch 03 正式匯入結果」，記錄匯入前 3145 筆、預計匯入 11 筆、實際成功匯入 11 筆、匯入後 3156 筆。
* `node scripts/auditSiteHealth.js` 通過。
* `node scripts/audit-reading-vocabulary.js` 通過，輸出仍顯示閱讀詞彙盤點中有 missing / confirm 項目，作為既有盤點狀態記錄。
* `node scripts/check-reading-ruby.js` 通過，但仍有既有 warning：`jp-reading-set-n4-019` 的 `一時半`、`jp-reading-set-n4-076` 的 `調べる`、`jp-reading-set-n4-102` 的 `十分前` 在部分 vocabulary / rubyTerms 中未出現在 title 或 passage。

## 2. 資料來源

本次實際讀取並確認目前日文閱讀資料來源為：

* `japaneseReadingQuestions.js`：以 `window.JAPANESE_READING_SETS = [...]` 定義閱讀資料。
* 每個 reading set 主要包含 `id`、`level`、`type`、`title`、`passage`、`passageKana`、`questions`、`vocabulary`、`grammarPoints`、`rubyTerms` 等欄位。
* 目前讀取結果為 105 組 reading sets、150 題 questions。
* 本次候選詞主要從 `title`、`passage`、`passageKana`、`questions`、`vocabulary`、`rubyTerms` 與既有 `docs/reading-vocabulary-audit.md` 盤點結果交叉確認。
* 正式單字庫比對來源為 `vocabulary.json`，目前 3156 筆。
* Batch 03 排除與匯入狀態比對來源為 `docs/reading-vocabulary-import-batch-03.md`。

## 3. 篩選原則

本批優先挑選 N5～N4 學習者實用且可獨立成詞的項目，例如日常生活、時間、交通、學校、活動、家庭、飲食、購物、天氣、身體與公共設施相關詞。

比對時採用以下原則：

1. 排除 `vocabulary.json` 中已存在完全相同 `word` 的詞。
2. 排除或標記 `vocabulary.json` 中已存在相同 `kana` 的詞。
3. 若中文意思與既有詞高度相近，於備註標記，避免後續產生重複測驗題。
4. 排除 Batch 03 已正式匯入的 11 詞，不重複列入候選。
5. 對 Batch 03 曾標記暫不匯入、需人工確認或疑似重複者，延續保守策略。
6. 標記是否已存在於閱讀資料自己的 `vocabulary` 或 `rubyTerms`，供後續人工確認是否只是閱讀輔助詞。
7. 排除人名、地名、店名、公司名、太長片語、標題式片語、助詞片段、語尾片段與假名或中文意思不明確者。
