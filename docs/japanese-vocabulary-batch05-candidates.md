## 3. 高優先修正候選

建議下一批實際修正優先處理 25 筆：

1. id 1428「輸出」：Batch 03 已修正 meaning，但目前中文例句仍用「出口」，容易回到舊誤解。
2. id 1455「調子」：初級常用詞，模板例句無法呈現「体／機械の調子」。
3. id 1457「場合」：N4 重要句型詞，應用「～の場合」呈現。
4. id 1473「係」：需以「掃除係」「受付の係」等具體職務呈現。
5. id 1477「審判」：中文「裁判」易歧義，需明確是比賽裁判。
6. id 1502「近所」：兼具「附近／鄰居」，現例句未呈現任一自然用法。
7. id 1504「角」：問路常用詞，應改為轉角場景。
8. id 1512「改札」：與「改札口」策略需確認，例句應使用車站場景。
9. id 1518「禁煙」：高頻標示詞，應改為「この店は禁煙です」。
10. id 1519「喫煙」：與禁煙成對，應補喫煙所或可否吸菸情境。
11. id 1520「会計」：餐廳／商店高頻用法，應以「会計をお願いします」呈現。
12. id 1521「注文」：應補「注文する」或點餐場景。
13. id 1523「返品」：購物情境明確，適合改自然問句。
14. id 1533「お釣り」：交易高頻詞，模板句學習效果弱。
15. id 1552「血」：身體／受傷場景常用，應改為「血が出る」。
16. id 1557「胸」：可改為症狀句「胸が痛いです」。
17. id 1558「腹」：語域與「お腹」區分需要人工確認。
18. id 1564「薬」：高頻基礎詞，應改「薬を飲む」。
19. id 1568「退院」：需呈現「退院する」，並確認詞性策略。
20. id 1573「親」：親屬核心詞，應避免「調べます」模板。
21. id 1595「授業」：學校核心詞，應改為上課開始等自然場景。
22. id 1615「停留所」：交通常用詞，需改為上下車場景。
23. id 1618「終電」：交通生活常用詞，應改為「間に合う」等自然搭配。
24. id 1620「乗り換え」：交通核心詞，應改為車站轉乘句。
25. id 1637「宿泊」：旅遊核心詞，需呈現「宿泊する」並確認詞性。

## 4. 建議暫不修改但需保留觀察的項目

1. id 1440「放送」：模板例句不佳，但 meaning 與假名本身正確；可與媒體類詞彙整批處理。
2. id 1442「記事」：例句模板化，但「報導；文章」語義可接受；可與新聞、報道、記者等同領域項目一起整理。
3. id 1452「香り」與 id 1454「匂い」：兩者都適合改自然例句，但需先決定是否要明確區分「香り」偏香氣、「匂い」可中性或臭味。

## 5. 後續建議

- Batch 05 實際修正時，建議先處理「模板句」中最影響學習的高頻生活詞，而不是一次性改完所有「について調べます」項目。
- 每筆修正應以自然生活、交通、購物、學校、旅遊或醫療場景呈現單字核心用法，避免「查詢關於某詞的資料」這類元語言或模板例句。
- `exampleKana` 必須完整對應 `example`，且保持全假名；若例句含片假名外來語，可依既有資料策略保留片假名或確認是否轉平假名，但不可混入漢字。
- `exampleMeaning` 應使用自然繁體中文，並準確對應日文例句；對多義詞只翻譯本例句所呈現的語義即可。
- 對「名詞」但例句自然會用「する」的詞，需先確認是否維持名詞、改為名詞/する動詞，或只在例句中使用「～をする」而不改詞性。
- 對地區用語差異大的中文（如車掌、停留所、番地、割引）應優先採用中性且清楚的譯法，必要時以分號補充。
- 對可能偏 N4 以上或旅遊補充詞的複合名詞，只建議標記人工確認，不宜在本批任務中刪除或降級。
- 實際修正後仍需執行 JSON、JavaScript 語法、站台健康、日文入口、中頁、view isolation、ruby rendering 與 git diff 檢查。
