# 方案 C Batch 05 閱讀單字候選清單

## 1. 本批任務目的

本文件整理「方案 C Batch 05」閱讀單字候選清單，目標是從目前專案實際載入的日文閱讀資料中，找出尚未正式收錄於 `vocabulary.json`、但可能適合 N5～N4 學習者補入正式日文單字庫的詞彙。本批只做候選清單準備與人工確認前置整理，不正式匯入任何單字，也不修改 `vocabulary.json`。

本次執行前確認：

1. `vocabulary.json` 目前總筆數為 3166 筆。
2. `docs/japanese-plan-c-batch04-post-import-check.md` 存在。
3. `docs/reading-vocabulary-import-batch-04.md` 已記錄 Batch 04 正式匯入結果，匯入後筆數為 3166 筆。
4. `node scripts/auditSiteHealth.js` 通過。
5. `node scripts/audit-reading-vocabulary.js` 通過；執行時會重產生 `docs/reading-vocabulary-audit.md`，本任務不保留該檔變更。
6. `node scripts/check-reading-ruby.js` 通過，但仍有既有 warning：`一時半`、`調べる`、`十分前` 相關文字未出現在對應文章本文或標題中。本批只記錄，不修正。

## 2. 資料來源

本次實際確認專案使用的閱讀資料來源如下：

1. `scripts/check-reading-ruby.js` 與 `scripts/audit-reading-vocabulary.js` 皆直接 `require` 專案根目錄的 `japaneseReadingQuestions.js`。
2. `japaneseReadingQuestions.js` 將資料掛載在 `window.JAPANESE_READING_SETS`。
3. 目前 `window.JAPANESE_READING_SETS` 共有 105 組閱讀資料、150 題閱讀題目。
4. 每組閱讀資料主要欄位包含 `id`、`level`、`type`、`title`、`passage`、`passageKana`、`questions`、`vocabulary`、`grammarPoints`、`rubyTerms`。
5. 本次候選詞來源以 `passage`、`passageKana`、`questions`、`vocabulary`、`rubyTerms` 的實際內容為主，並參考 `docs/reading-vocabulary-audit.md` 與 Batch 03、Batch 04 文件中已處理或已排除項目，避免重複納入。

## 3. 篩選原則

本批優先挑選下列詞彙：

1. 適合 N5～N4 學習者的日常生活、交通、學校、購物、天氣、身體、時間、飲食與公共設施主題詞。
2. 在閱讀文章、題目、`vocabulary` 或 `rubyTerms` 中出現，且假名、中文意思、詞性可合理判斷的詞。
3. 尚未在 `vocabulary.json` 中出現完全相同 `word`，且未發現相同 `kana` 的既有正式詞。
4. 未在 Batch 03 或 Batch 04 中正式匯入，且未被明確標記為不應重複處理的詞。
5. 優先選短詞或語義固定的複合詞，避免太長片語與文章標題型表現。

本批排除或標記為暫不匯入的詞彙包含：

1. 人名、地名、公司名、店名或特定路線名。
2. 太長或太像文章標題的片語。
3. 助詞、語尾、接續片段或活用形本身。
4. 假名或中文意思不明確的詞。
5. 已存在於 `vocabulary.json` 的詞。
6. Batch 03、Batch 04 已正式匯入或已明確排除的詞，例如 `温める` 已存在不需匯入、`誘い` 暫不進本批。
