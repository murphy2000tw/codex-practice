## 3. 建議暫不修改清單

1. 「お菓子」與「菓子」：兩者雖近義，但「お菓子」是常見日常說法，「菓子」較偏書面、商品分類或複合詞，建議保留並觀察後續是否需要補充語域說明。
2. 「お茶」：目前未與單獨「茶」重複；作為 N5 常用形式有學習價值，建議先保留。若未來新增「茶」，再區分「ちゃ」與「おちゃ」的語域及用法。
3. 「お金」：目前未與「金（かね）」重複；「お金」是常用形式，建議先保留。若未來新增「金」，需避免與「金（きん，黃金）」混淆。
4. 「弁当」：目前未與「お弁当」重複；建議保留。若未來新增「お弁当」，再補充禮貌或口語常用差異。
5. 「暖かい」與「温かい」：同讀音但常用語境不同，前者較常用於天氣、氣候或整體溫暖感，後者較常用於物體、食物、感情等溫暖，建議保留並觀察 meaning 是否需強化。
6. 「早い」與「速い」：同讀音但語義可區分；「早い」偏時間早，「速い」偏速度快，建議保留並在後續需要時強化 meaning 差異。

## 4. Batch 03 實際修改原則

- 名詞與「する」動詞可以並存，但需明確分工：名詞項的 `partOfSpeech` 建議維持「名詞」，動詞項的 `partOfSpeech` 建議維持「動詞」，避免在已有獨立動詞項時再使用「名詞/する動詞」。
- 若「名詞」與「する」動詞並存且 level 不一致，實際修改前應逐筆確認分級是否有教學依據；若無明確理由，建議調整為同級或在修正紀錄中說明保留原因。
- 對自然度較低的「する」動詞，應確認是否改為更自然的片語或基礎動詞，例如「仕事をする」、「会議をする」、「試験を受ける」、「申し込む」。若資料格式不適合收片語，則需在 meaning 中避免誤導。
- 「お＋名詞」的禮貌詞或美化語不應自動刪除；若其為 N5 / N4 常用形式，應依常用度、語域與是否有單獨名詞項來決定是否並存。
- 漢字與假名表記可以並存，但需有清楚理由，例如常用假名、學習分級或表記差異；若只是同 kana、同 meaning、同詞性重複，應優先考慮刪併或統一表記。
- 同讀音不同漢字的詞不應僅因 kana 相同就合併；例如「暖かい / 温かい」、「早い / 速い」需保留語義差異，並用 meaning 說明使用場景。
- meaning 應避免過度模糊或詞性不一致：形容詞建議使用「～的」，動詞使用動作義，名詞使用名詞義；對易混淆詞應補充語境，如「輸出」需避免被誤解為建築物「出口」。
- `partOfSpeech` 應盡量使用既有分類並保持一致，例如「名詞」、「動詞」、「い形容詞」、「な形容詞」；除非資料庫沒有對應獨立動詞項，否則避免混用「名詞/する動詞」。
- 實際修改 `vocabulary.json` 時，不應刪除任何單字，除非另有明確任務指示；若需要合併或移除，應先產生人工確認清單。
- 每次實際修改後都應重新執行 JSON 格式檢查、JavaScript 語法檢查、站台健康檢查與日文單字相關檢查，並確認沒有非預期檔案變更。
