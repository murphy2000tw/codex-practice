## 4. Batch 05 實際修改原則

- `example` 必須自然，且實際使用該單字；避免「〜について調べます」或談論單字本身的元語言模板。
- `exampleKana` 必須完整對應 `example`，不得漏字、漏助詞或改變句意，且不得混入漢字。
- `exampleMeaning` 必須自然、準確，符合繁體中文語感，並只翻譯該例句實際呈現的語義。
- 例句難度應符合 N5 / N4 等級，優先使用生活、交通、購物、學校、旅遊、身體症狀等初級場景。
- 若例句太書面、太抽象，應改成生活化例句；若詞彙本身偏書面或偏專門，應在修正理由中標明並保留人工確認。
- 若同一類型用法可保留，例如名詞搭配「する」或複合名詞作旅遊補充詞，需在理由中說明保留原因與後續確認點。
- 多義詞不必在單一例句中涵蓋所有 meaning，但例句與中文翻譯必須對應同一語義，避免造成歧義。
- 對「名詞」但自然例句會使用「する」的詞，實際修改前應確認是否只改例句，或同步調整 `partOfSpeech` 為「名詞/する動詞」。
- 中文譯法若有地區差異，例如「車掌」「停留所」「番地」「割引」，應採清楚、中性的繁中用語，必要時以分號補充。
- 不要刪除任何單字，不要改動既有 id，不要因例句修正而移除詞條。
- 不要修改 `script.js` 或現有功能；Batch 05 實際修正應限於經人工確認的 `vocabulary.json` 欄位。
- 實際修改後必須重新執行 JSON、JavaScript 語法、站台健康與日文單字相關檢查，並確認 `git diff --check` 無空白錯誤。
