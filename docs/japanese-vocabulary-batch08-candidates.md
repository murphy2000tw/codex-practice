## 5. 後續建議

- Batch 08 實際修正時，仍應遵守 Batch 03～07 原則：不刪除單字、不變更既有 id、不因表記差異合併詞條、不修改 script.js。
- 本批重點建議放在 meaning 與 exampleMeaning 品質，不要同時展開新的外來語 kana 全域策略。
- 優先修正「中文同形反寫但缺乏解釋」的項目，例如「場所」「初診」「身分確認」。
- 對行政、醫療、購物、租屋類詞彙，應採台灣學習者易懂的繁中用語，例如「繳費窗口」「承辦人」「第一次看診」「到貨」。
- 若 meaning 為名詞，避免寫成完整動作句；若例句使用「する」動作，需確認 partOfSpeech 是否應改為「名詞/する動詞」。
- exampleMeaning 應直接翻譯日文例句，不要使用「查詢關於～的資料」「確認關於～的事」作為大量模板；例句應呈現該字核心搭配。
- 多義詞應在 meaning 中保留主要義並讓例句對應其中一個核心用法；若中文只呈現部分意思，應避免與 exampleMeaning 互相矛盾。
- 同類詞需批次統一：客/お客様/観光客、控え/写し/コピー、支払/会計/窓口、特売品/セール品/割引、受付/受付時間/受付番号。
- 對 Batch 07 已保留觀察的「受付時間」「土産店」「机上」，若 Batch 08 要實修，應先確認詞形自然度與 N4 學習適切性，再同步改 meaning、example、exampleKana、exampleMeaning。
