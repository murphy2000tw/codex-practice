## 4. Batch 06 實際修改原則

- 不刪除單字。
- 不改變既有 id。
- 不因同義或近義就合併詞條。
- partOfSpeech 必須和 word、meaning、example 實際用法一致。
- meaning 必須清楚，避免只寫「做」「事」「地方」「東西」「情況」等過度模糊說明。
- 名詞與「する」動詞若可並存，需清楚區分詞性與意思；若例句使用「する」而資料庫沒有獨立動詞詞條，才優先考慮「名詞/する動詞」。
- い形容詞、な形容詞、副詞、接續詞、連體詞、助詞不可混用。
- level 調整需有明確理由，不可只憑主觀感覺調整；若只是偏書面、偏旅遊、偏公告用語，應先在理由中標明並保留人工確認。
- example / exampleKana / exampleMeaning 若一起修正，需保持三者一致；exampleKana 必須完整對應 example，且全假名化。
- 例句應呈現單字的實際用法，避免「〜について調べます」或談論單字本身的元語言模板。
- 多義詞不必在單一例句涵蓋所有 meaning，但 example 與 exampleMeaning 必須對應同一語義。
- 時間詞與方位詞若只是後接「に」形成副詞性，通常不應輕易標為「副詞」。
