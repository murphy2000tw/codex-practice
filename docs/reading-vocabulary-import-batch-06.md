# 方案 C Batch 06 閱讀單字候選清單

## 1. 本批任務目的

本批任務定位為「閱讀文章候選詞整理」，只從目前專案實際載入的日文閱讀資料中整理下一批可能適合加入正式日文單字庫的候選詞，供後續人工確認使用。本次不正式匯入任何單字，不修改 `vocabulary.json`，也不修改日文閱讀頁、日文單字頁、日文測驗頁、英文頁面或 Batch 03～Batch 05 文件。

本次執行前確認：

1. `vocabulary.json` 目前總筆數為 3177 筆。
2. `docs/japanese-plan-c-batch05-post-import-check.md` 存在。
3. `docs/reading-vocabulary-import-batch-05.md` 已記錄 Batch 05 正式匯入結果與後續核對內容。
4. `node scripts/auditSiteHealth.js` 通過。
5. `node scripts/audit-reading-vocabulary.js` 通過；執行時會重產生 `docs/reading-vocabulary-audit.md`，本任務不保留該檔變更。
6. `node scripts/check-reading-ruby.js` 通過，但仍有既有 warning：`一時半`、`調べる`、`十分前` 相關文字未出現在對應文章本文或標題中。本批只記錄，不修正。

## 2. 資料來源

本次實際讀取並確認目前日文閱讀資料來源如下：

1. `script.js` 會讀取 `window.JAPANESE_READING_SETS`，並以 `readingSet` 的 `title`、`passage`、`passageKana`、`questions`、`vocabulary`、`rubyTerms` 等欄位呈現閱讀文章、假名輔助、題目、重要單字與 ruby 標註。
2. `scripts/audit-reading-vocabulary.js` 與 `scripts/check-reading-ruby.js` 皆直接載入專案根目錄的 `japaneseReadingQuestions.js`。
3. `japaneseReadingQuestions.js` 以 `window.JAPANESE_READING_SETS = [...]` 定義目前閱讀資料；本次實際檢查結果為 105 組 reading sets、150 題 questions。
4. 本次候選整理實際比對 `passage`、`passageKana`、`questions`、`vocabulary`、`rubyTerms`，並參考 `docs/reading-vocabulary-audit.md` 的盤點結果與 Batch 03、Batch 04、Batch 05 文件中已匯入、暫緩、需人工再確認或不建議匯入的詞。
5. 正式單字庫比對來源為 `vocabulary.json`，目前維持 3177 筆。

## 3. 篩選原則

本批優先考慮 N5～N4 學習者可能在生活、交通、學校、公共設施、時間、購物、天氣與社區情境中實際遇到的詞。篩選時同時排除或標記以下類型：

1. 人名、地名、站名、店名、公司名或其他專有名詞。
2. 太長、太像文章標題或情境描述的片語。
3. 助詞、語尾、接續片段或未還原成辭書形的動詞變化。
4. 假名、中文意思或詞性不夠明確的詞。
5. 與正式單字庫已有 `word`、`kana` 或中文意思高度重複的詞。
6. Batch 03、Batch 04、Batch 05 已正式匯入、已排除、已標記暫不匯入或需要人工再確認的詞。
7. 特別避免重複推進 Batch 04 / Batch 05 已處理詞，例如 `温める`、`誘い`、`変更`、`見学`、`整理`、`二週間`；若出現在盤點中，只列為已處理或不進本批。

## 7. 與正式單字庫比對結果

2. 本次候選總表 34 筆中，3 筆已有完全相同正式詞或可由正式庫辭書形覆蓋：`迷う`、`温める`、`調べる`，均不新增。

4. 本次也檢查是否已有相同 `kana` 或中文意思相近詞：`佐藤` 與既有 `砂糖` 同假名，排除；`整理` 與既有 `整理する` 同詞根，排除；`誘い` 與既有邀請相關詞根高度相近，且前批已處理，排除。
